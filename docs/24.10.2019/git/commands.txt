Основные команды git:
git init - инициализировать в текущей папке репозиторий git (локальный, т.е. размещенный на текущем компьютере)
git status - посмотреть состояние репозитория (какие файлы изменены, какие добавлены на отслеживание, какие не добавлены)
git add - добавление файлов для отслеживания, например:
git add . - добавить все файлы и папки, которые еще не добавлены
git add -u - добавить все измененные файлы
git add path/to/file.html - добавить на отслеживание файл file.html, находящийся в папке path/to относительно текущего вашего месторасположения в файловой системе
git commit - создание контрольной точки
git log - просмотр списка контрольных точек
git commit -m'комментарий' - создание контрольной точки сразу с комментарием (записывается в кавычках)
git checkout - команда переключения между контрольными точками, ветками, откат до состояния файла на момент создания текущей контрольной точки
git checkout path/to/file.html - отменить изменения в файле file.html, находящийся в папке path/to относительно текущего вашего месторасположения в файловой системе и вернуть его на состояние текущей контрольной точки
git checkout 994adf - переключиться на контрольную точку с id, начинающимся с 994adf
git checkout master - переключиться на ветку master
git branch some_name -  создать ветку с названием some_name (клонируется от текущей ветки)
git branch - просмотреть список веток

git remote -v - посмототреть список подключенных к репозиторию удаленных файлов
git remote add repoName repoAddress - подключить удаленный репозиторий с адресом "repoAddress" к локальному репозиторию и назначить ему имя "repoName"
git remote delete repoName - удалить из локального репозитория информацию об удаленном репозитории repoName (с самим удаленным репозиторием ничего не происходит)
git pull repoName branchName - получить содержимое ветки "branchName" из репозитория "repoName"
git push repoName branchName - отправить изменения ветки "branchName" в репозиторий "repoName" (если такой ветки не было - создать ее)
